# 04 · Agent 的要素

## 核心五要素（本项目全都有）

| 要素 | 角色 | 本项目对应 |
|---|---|---|
| **模型（大脑）** | 理解决策，Agent 的智能来源 | `gpt-4o-mini`（`agent.js:49`） |
| **工具（手和眼）** | 感知世界（查天气）、改变世界（写库、发邮件） | `get_weather`（`agent.js:4-23`） |
| **循环（Loop）** | 决策→执行→回流→再决策 | `for` 循环（`agent.js:47-90`），详见 [02](./02-the-loop.md) |
| **上下文（Messages）** | 累积的工作台，模型每轮看全量历史 | `messages.push(...)`（`agent.js:60,84`） |
| **指令（System Prompt）** | 角色、规则、边界的"岗位说明书" | `agent.js:42` |

**工具定义三要素**：`name`（叫什么）、`description`（干什么——模型靠它判断何时用）、`parameters`（怎么调）。

## 生产级 Agent 的扩展要素（本项目待升级方向）

| 扩展要素 | 作用 | 本项目现状 |
|---|---|---|
| 记忆（Memory） | 跨对话记住用户偏好与历史 | ❌ 每次全新 messages |
| 规划（Planning） | 大任务拆成子任务 | ❌ 单步决策 |
| 多工具/工具选择 | 从工具箱挑选工具 | ⚠️ 只有 1 个工具 |
| 反思（Reflection） | 自查输出质量并修正 | ❌ |
| 人机协同（HITL） | 危险操作前人类确认 | ❌ 全自动 |
| 护栏（Guardrails） | 防失控、限资源 | ✅ 雏形（5 轮上限、错误捕获） |

## 一句话总结

> **Agent = 模型在循环里，基于上下文和指令，自主决定调工具还是给答案。**

判断一个程序是不是 Agent，就看一条：**流程的分叉是模型决定的，还是代码写死的**。
